## 🔍 故障排查

### 1. 后端启动失败

**错误：`EADDRINUSE: address already in use`**

解决：
```bash
# 停止所有node进程
taskkill /F /IM node.exe

# 重新启动后端
cd c:/Users/DELL/CodeBuddy/route1/backend
node simple-server.js
```

### 2. 数据库连接失败

**错误：`Access denied for user 'root'@'localhost'**

检查：
1. MySQL是否运行
2. 密码是否为 `root`
3. 数据库 `campus_learning_platform` 是否存在

修复：
```bash
cd c:/Users/DELL/CodeBuddy/route1/backend
node init-users.js
```

### 3. 前端无法连接后端

**检查1：后端是否真的在运行**
```bash
# 测试后端API
powershell -Command "Invoke-WebRequest http://localhost:3006/api/health"
```

应该返回：`{"status":"ok","message":"Server is running"}`

**检查2：浏览器控制台错误**
- 按F12打开开发者工具
- 查看Console标签
- 查看Network标签，看请求是否失败

常见错误：
- `net::ERR_CONNECTION_REFUSED` - 后端没启动
- `502 Bad Gateway` - 后端启动但无法连接
- `401 Unauthorized` - 登录token无效

### 4. 代理配置错误

确保 `frontend/vite.config.ts` 配置正确：
```typescript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:3006',  // 必须匹配后端端口
      changeOrigin: true
    }
  }
}
```

### 5. 端口冲突

如果其他服务占用了端口：

**查看占用端口的进程：**
```bash
# 查看3006端口
netstat -ano | findstr :3006

# 查看5173端口
netstat -ano | findstr :5173
```

**修改端口：**
- 后端：修改 `backend/simple-server.js` 中的 `PORT`
- 前端：修改 `frontend/vite.config.ts` 中的 `proxy.target`

## 🎯 常见问题

### Q: 为什么点击功能都显示"请求失败"？

A: 最可能的原因是**后端服务没有运行**。

解决：
1. 检查是否有后端终端窗口打开
2. 如果没有，运行 `start-all.bat`
3. 或者手动启动后端：
   ```bash
   cd c:/Users/DELL/CodeBuddy/route1/backend
   node simple-server.js
   ```

### Q: 登录成功了，但其他功能还是失败？

A: 可能是token过期或无效。

解决：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 退出登录重新登录
3. 检查localStorage中是否有token

### Q: 有些功能能用，有些不能用？

A: 某些API可能没有实现或数据表为空。

解决：
1. 查看浏览器控制台错误信息
2. 查看后端终端日志
3. 检查数据库中是否有对应数据
